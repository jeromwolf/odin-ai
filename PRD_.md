# Product Requirements Document (PRD)
## Flux Marketplace - 공공조달 플랫폼

### 1. 프로젝트 개요

**제품명**: Flux Marketplace  
**프로젝트 기간**: 2024년 - MVP 6개월, 정식 출시 12개월  
**목표**: 나라장터 입찰정보의 AI 기반 분석 및 맞춤형 추천 서비스

### 2. 문제 정의

**현재 상황**:
- 매일 수백 개의 입찰공고가 나라장터에 게시됨
- 기업들이 수동으로 공고를 확인하여 시간 소모 및 놓치는 기회 발생
- RFP 문서(HWP/PDF) 분석에 많은 시간과 인력 투입
- 수주 가능성 판단의 어려움

**해결하려는 문제**:
- 입찰정보 모니터링 자동화
- AI 기반 수주 가능성 예측
- 맞춤형 프로젝트 추천
- RFP 문서의 신속한 분석 및 요약

### 3. 타겟 사용자

**Primary Users**:
- 중소기업 사업개발팀
- IT 서비스 회사
- 건설/엔지니어링 회사
- 컨설팅 회사

**User Personas**:
1. **김사업 팀장** (40세, IT서비스 회사)
   - 매일 나라장터 확인하는 업무 담당
   - 시간 절약과 효율성 중시
   - 기술적 배경 보통

2. **박기획 과장** (35세, 건설회사)
   - 공공공사 입찰 전문
   - 수주율 향상이 핵심 목표
   - 데이터 분석에 관심 있음

### 4. 핵심 기능 (Core Features)

#### 4.1 데이터 수집 및 처리
- **나라장터 실시간 크롤링**
  - 입찰공고, 수의계약, 긴급조달 정보
  - 30분마다 자동 업데이트
  - 중복 제거 및 데이터 정규화

- **문서 처리 시스템**
  - HWP → PDF 자동 변환
  - PDF 텍스트 추출 및 파싱
  - 핵심 정보 자동 분류 (예산, 기간, 요구사항)

#### 4.2 AI 분석 시스템
- **RAG 기반 문서 검색**
  - RFP 문서의 벡터화 및 임베딩
  - 자연어 질의응답 시스템
  - 유사 과제 추천

- **수주 가능성 예측**
  - 회사 프로필 vs 프로젝트 매칭 스코어
  - 과거 입찰 결과 학습 기반 예측
  - 경쟁률 및 성공 확률 계산

#### 4.3 사용자 경험
- **맞춤형 대시보드**
  - 개인화된 프로젝트 피드
  - 수주 가능성별 색상 구분
  - 실시간 알림 시스템

- **상세 분석 리포트**
  - RFP 요약 및 핵심 포인트 추출
  - 경쟁사 분석
  - 제안서 작성 가이드

#### 4.4 통계 및 분석
- **시장 동향 분석**
  - 분야별 입찰 트렌드
  - 예산 규모별 분포
  - 발주기관별 패턴 분석

- **성과 분석**
  - 개별 회사 수주율 분석
  - 업계 벤치마킹
  - ROI 계산 및 개선 제안

### 5. 기술 스택

#### 5.1 Backend
```yaml
Framework: FastAPI (Python 3.11+)
Database: PostgreSQL + Redis
Queue: Celery + Redis
AI/ML: 
  - OpenAI GPT-4 (또는 Claude)
  - LangChain (RAG)
  - Pinecone/Weaviate (Vector DB)
  - scikit-learn (예측 모델)
```

#### 5.2 Document Processing
```yaml
HWP Processing: 
  - hwp5 라이브러리
  - LibreOffice headless 변환
  - OCR: Tesseract (이미지 문서 대응)
PDF Processing:
  - PyPDF2, pdfplumber
  - 테이블 추출: tabula-py
```

#### 5.3 Infrastructure
```yaml
Containerization: Docker + Kubernetes
Cloud: AWS/GCP (Auto-scaling)
Monitoring: Grafana + Prometheus
CI/CD: GitHub Actions
```
