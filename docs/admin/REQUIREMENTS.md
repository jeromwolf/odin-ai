# 관리자 웹 시스템 요구사항 정의서

> ODIN-AI 관리자 대시보드 상세 요구사항
> 작성일: 2025-10-02

---

## 1. 배치 모니터링 요구사항

### 1.1 실시간 배치 실행 상태 확인

#### 기능 설명
관리자가 현재 실행 중인 배치 프로그램의 상태를 실시간으로 확인할 수 있어야 합니다.

#### 상세 요구사항
- **실행 중인 배치 표시**
  - 현재 실행 중인 배치 타입 (collector/downloader/processor/notification)
  - 시작 시간
  - 경과 시간
  - 진행률 (처리 중인 항목 수 / 전체 항목 수)
  - 상태: `실행중`, `대기중`, `없음`

- **최근 완료된 배치 표시**
  - 최근 5개 배치 실행 결과
  - 실행 시간, 소요 시간, 성공/실패 여부

- **실시간 업데이트**
  - 5초마다 자동 갱신 (WebSocket 또는 Polling)
  - 수동 새로고침 버튼

#### 수용 기준
- [ ] 실행 중인 배치가 5초 이내에 화면에 반영됨
- [ ] 배치 상태가 정확하게 표시됨 (실제 상태와 일치)
- [ ] 네트워크 지연 시에도 UI가 멈추지 않음

---

### 1.2 배치 실행 이력 조회

#### 기능 설명
과거 배치 실행 이력을 조회하고 분석할 수 있어야 합니다.

#### 상세 요구사항
- **필터링 옵션**
  - 날짜 범위 선택 (DateRangePicker)
  - 배치 타입 선택 (전체/collector/downloader/processor/notification)
  - 상태 선택 (전체/성공/실패/실행중)

- **테이블 컬럼**
  | 컬럼명 | 설명 | 정렬 가능 |
  |--------|------|-----------|
  | ID | 실행 ID | ✅ |
  | 배치 타입 | collector/downloader 등 | ✅ |
  | 시작 시간 | YYYY-MM-DD HH:mm:ss | ✅ |
  | 종료 시간 | YYYY-MM-DD HH:mm:ss | ✅ |
  | 소요 시간 | 초 단위 (분:초 형식) | ✅ |
  | 처리 건수 | 성공/전체 | ✅ |
  | 상태 | 성공/실패 (색상 표시) | ✅ |
  | 액션 | 상세보기 버튼 | - |

- **페이지네이션**
  - 페이지당 20개 항목
  - 페이지 번호 표시
  - 이전/다음 버튼

#### 수용 기준
- [ ] 필터 적용 시 1초 이내 결과 표시
- [ ] 10,000개 이상의 이력도 정상 조회 가능
- [ ] 테이블 정렬 기능 정상 작동

---

### 1.3 배치별 처리 통계

#### 기능 설명
각 배치별 처리 성공률, 평균 처리 시간 등의 통계를 확인할 수 있어야 합니다.

#### 상세 요구사항
- **통계 지표**
  - 총 실행 횟수
  - 성공 횟수 / 실패 횟수
  - 성공률 (%)
  - 평균 처리 시간
  - 최대 처리 시간
  - 최소 처리 시간
  - 평균 처리 건수

- **배치 타입별 통계**
  - Collector: API 호출 성공률, 평균 수집 건수
  - Downloader: 다운로드 성공률, 파일 타입별 성공률
  - Processor: 문서 처리 성공률, 태그 생성 건수
  - Notification: 알림 발송 성공률, 매칭 건수

- **시각화**
  - 성공/실패 비율 파이 차트
  - 일별 처리 건수 바 차트
  - 처리 시간 추이 라인 차트

#### 수용 기준
- [ ] 통계 데이터가 정확함 (DB 데이터와 일치)
- [ ] 차트가 즉시 렌더링됨 (로딩 스피너 표시)
- [ ] 날짜 범위 변경 시 통계가 즉시 업데이트됨

---

### 1.4 실행 시간 분석

#### 기능 설명
배치 실행 시간을 분석하여 성능 개선 포인트를 파악할 수 있어야 합니다.

#### 상세 요구사항
- **시간 분석 지표**
  - 평균 실행 시간 (전체/배치별)
  - 실행 시간 분포 (히스토그램)
  - 최근 30일 실행 시간 추이
  - 가장 느린 실행 TOP 10

- **단계별 시간 분석**
  - Collector: API 호출 시간, 파싱 시간
  - Downloader: 다운로드 시간 (파일 타입별)
  - Processor: 문서 처리 시간, 정보 추출 시간

- **성능 경고**
  - 평균 대비 2배 이상 느린 실행 하이라이트
  - 실행 시간 임계값 설정 (예: Collector > 10분)

#### 수용 기준
- [ ] 시간 분석 차트가 정확하게 표시됨
- [ ] 느린 실행 건에 대한 경고가 표시됨
- [ ] 단계별 시간 분해 데이터 제공

---

### 1.5 에러 발생 현황 및 상세 로그

#### 기능 설명
배치 실행 중 발생한 에러를 추적하고 상세 로그를 확인할 수 있어야 합니다.

#### 상세 요구사항
- **에러 현황 대시보드**
  - 오늘 발생한 에러 건수
  - 최근 7일 에러 추이 차트
  - 에러 타입별 분포
  - 가장 많이 발생한 에러 TOP 5

- **에러 상세 로그**
  - 에러 발생 시간
  - 배치 타입 및 실행 ID
  - 에러 메시지 (전체 스택 트레이스)
  - 에러 컨텍스트 (처리 중이던 항목 정보)
  - 재시도 여부 및 결과

- **에러 검색 및 필터링**
  - 키워드 검색 (에러 메시지 내 검색)
  - 에러 타입 필터 (API 에러, 파싱 에러, DB 에러 등)
  - 날짜 범위 필터

- **에러 알림 설정**
  - 특정 에러 발생 시 관리자 이메일 발송
  - 에러 임계값 설정 (예: 1시간 내 10건 이상)

#### 수용 기준
- [ ] 에러 로그가 실시간으로 반영됨
- [ ] 스택 트레이스 전체가 읽기 쉽게 표시됨
- [ ] 에러 검색 기능이 빠르게 작동 (1초 이내)

## 2. 시스템 상태 모니터링 요구사항

### 2.1 서버 리소스 사용량

#### 기능 설명
서버의 CPU, 메모리, 디스크 사용량을 실시간으로 모니터링할 수 있어야 합니다.

#### 상세 요구사항
- **실시간 메트릭**
  - CPU 사용률 (%)
  - 메모리 사용률 (%) 및 사용량 (GB)
  - 디스크 사용률 (%) 및 사용량 (GB)
  - 네트워크 트래픽 (In/Out MB/s)

- **시각화**
  - 실시간 라인 차트 (최근 1시간 데이터)
  - 게이지 차트 (현재 사용률)
  - 임계값 표시 (70% 경고, 90% 위험)

- **히스토리 조회**
  - 최근 24시간 / 7일 / 30일
  - CSV 다운로드 기능

- **알림 설정**
  - CPU > 90% 지속 시 알림
  - 메모리 > 85% 지속 시 알림
  - 디스크 > 80% 시 알림

#### 수용 기준
- [ ] 메트릭이 5초마다 업데이트됨
- [ ] 차트가 부드럽게 애니메이션됨
- [ ] 임계값 초과 시 색상 변경 (빨간색)

---

### 2.2 데이터베이스 연결 상태

#### 기능 설명
PostgreSQL 데이터베이스의 연결 상태 및 성능을 모니터링할 수 있어야 합니다.

#### 상세 요구사항
- **연결 상태**
  - 연결 상태: 정상/비정상
  - 활성 연결 수
  - 최대 연결 수
  - 대기 중인 쿼리 수

- **성능 지표**
  - 평균 쿼리 응답 시간
  - 느린 쿼리 (1초 이상)
  - 쿼리 실행 횟수 (분당)
  - 캐시 히트율

- **데이터베이스 크기**
  - 전체 DB 크기
  - 테이블별 크기 (상위 10개)
  - 인덱스 크기

#### 수용 기준
- [ ] DB 상태가 정확하게 표시됨
- [ ] 느린 쿼리가 즉시 감지됨
- [ ] DB 크기 정보가 정확함

---

### 2.3 API 응답 시간 모니터링

#### 기능 설명
사용자 웹 API의 응답 시간을 모니터링하여 성능 저하를 조기에 발견할 수 있어야 합니다.

#### 상세 요구사항
- **엔드포인트별 응답 시간**
  - 평균 응답 시간
  - 최대 응답 시간
  - P95, P99 응답 시간
  - 요청 횟수

- **응답 시간 추이**
  - 최근 1시간 라인 차트
  - 엔드포인트별 색상 구분

- **느린 API 경고**
  - 평균 응답 시간 > 1초
  - P95 > 2초
  - 에러율 > 5%

- **API 상태 코드 분포**
  - 2xx (성공)
  - 4xx (클라이언트 에러)
  - 5xx (서버 에러)

#### 수용 기준
- [ ] 모든 API 엔드포인트가 추적됨
- [ ] 느린 API가 하이라이트됨
- [ ] 응답 시간 차트가 정확함

---

### 2.4 알림 발송 현황

#### 기능 설명
이메일 알림 발송 현황을 모니터링할 수 있어야 합니다.

#### 상세 요구사항
- **발송 통계**
  - 오늘 발송된 알림 수
  - 성공/실패 건수
  - 성공률 (%)
  - 대기 중인 알림 수

- **알림 타입별 현황**
  - 입찰 매칭 알림
  - 마감 임박 알림
  - 일일 다이제스트
  - 시스템 알림

- **발송 실패 분석**
  - 실패 원인별 분류
  - 실패한 이메일 주소 목록
  - 재시도 현황

- **발송 시간 분석**
  - 평균 발송 시간
  - 시간대별 발송 건수

#### 수용 기준
- [ ] 발송 통계가 정확함
- [ ] 실패 원인이 명확하게 표시됨
- [ ] 재시도 기능이 정상 작동함

## 3. 사용자 관리 요구사항

### 3.1 사용자 목록 조회

#### 기능 설명
가입된 사용자 목록을 조회하고 관리할 수 있어야 합니다.

#### 상세 요구사항
- **사용자 목록 테이블**
  | 컬럼 | 설명 |
  |------|------|
  | ID | 사용자 ID |
  | 이메일 | 로그인 이메일 |
  | 이름 | 사용자 이름 |
  | 구독 플랜 | Free/Basic/Pro/Enterprise |
  | 가입일 | YYYY-MM-DD |
  | 마지막 로그인 | YYYY-MM-DD HH:mm |
  | 상태 | 활성/비활성 |

  - 이메일 검색
  - 이름 검색
  - 회사명 검색

- **필터링**
  - 구독 플랜별 필터
  - 활성/비활성 상태 필터
  - 가입일 범위 필터

- **정렬**
  - 가입일순
  - 마지막 로그인순
  - 이름순

#### 수용 기준
- [ ] 검색 결과가 즉시 표시됨
- [ ] 필터링이 정확하게 작동함
- [ ] 페이지네이션이 부드럽게 작동함

개별 사용자의 상세 정보를 조회할 수 있어야 합니다.

#### 상세 요구사항
- **기본 정보 탭**
  - 이메일, 이름, 회사, 직책
  - 전화번호

- **알림 규칙 탭**
  - 등록된 알림 규칙 목록
  - 키워드, 가격 범위, 지역
  - 알림 발송 통계

- **북마크 탭**
  - 저장한 입찰 목록
  - 북마크 날짜
  - 입찰 상태

#### 수용 기준
- [ ] 모든 탭이 즉시 로드됨
- [ ] 데이터가 정확하게 표시됨
- [ ] 탭 전환이 부드러움

#### 기능 설명

#### 수용 기준
- [ ] 계정 관리 액션이 즉시 반영됨
- [ ] 변경 이력이 정확하게 기록됨
- [ ] 관리자 활동 로그가 자동 기록됨

## 4. 통계 및 분석 요구사항

### 4.1 일별/주별/월별 입찰 수집 통계
